# PUML Builder

드래그앤드롭으로 UML 다이어그램을 구성하고 **PlantUML 코드를 실시간으로 내보내는** 단일 HTML 도구입니다. 설치·빌드·외부 의존성이 없어 파일 하나만 열면 바로 동작합니다.

지원하는 다이어그램: **클래스 · 시퀀스 · 로버스트니스 · 스테이트**

## 사용법

`index.html`을 브라우저로 열면 됩니다.

- 상단 탭에서 다이어그램 종류를 전환합니다.
- 클래스·스테이트·로버스트니스: 왼쪽 도구로 노드를 추가하고 드래그해 배치한 뒤, 연결 도구를 누르고 시작·대상 노드를 차례로 클릭해 선으로 잇습니다. 노드를 클릭하면 오른쪽에서 이름·속성·메서드를 편집합니다.
- 시퀀스: 오른쪽 패널에서 참여자와 메시지를 추가하고, 참여자 박스를 캔버스에서 드래그해 순서를 바꿉니다.
- 오른쪽 아래 코드 패널이 실시간으로 갱신됩니다. `복사`로 코드를 가져가거나, `미리보기 ↗`로 plantuml.com에서 렌더링 결과를 확인합니다.
- 상단 `불러오기` / `.puml 내려받기`로 파일을 주고받습니다. 작업 내용은 브라우저에 자동 저장됩니다.

## 동작 방식 / 한계

- **단방향**: GUI로 만든 구성을 PlantUML 코드로 출력합니다. 코드를 다시 GUI로 역파싱하지는 않습니다. PlantUML은 좌표를 코드에 담지 않고 레이아웃을 렌더링 엔진(Graphviz)이 결정하므로, 위치 정보를 코드로 되돌리는 양방향 편집은 구조적으로 어렵습니다.
- **로버스트니스**: PlantUML에 전용 문법이 없어 `actor` / `boundary` / `control` / `entity` 키워드로 ICONIX 스타일 아이콘을 출력합니다.
- **자동 저장**은 브라우저 `localStorage` 기준이라 브라우저·도메인별로 따로 보관되며, 서버에 공유되지 않습니다(공동 편집 아님).
- **미리보기**는 코드를 공개 plantuml.com 서버로 전송해 렌더링합니다. 외부 전송이 곤란한 환경이면 해당 호출 주소를 사내 PlantUML 서버로 바꿔 쓰면 됩니다.

## GitHub Pages로 배포하기

1. 이 저장소를 GitHub에 올립니다.
2. 저장소의 `Settings → Pages`로 이동합니다.
3. Source를 `Deploy from a branch`, 브랜치를 `main` / 폴더를 `/ (root)`로 지정하고 저장합니다.
4. 잠시 후 `https://<사용자명>.github.io/<저장소이름>/` 에서 접속됩니다.

빌드 단계가 없으므로 `index.html`만 루트에 있으면 그대로 게시됩니다.
